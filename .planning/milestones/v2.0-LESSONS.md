- 파일 동일성 검증: `diff -q source target` 조합 — 마이그레이션/복사 작업의 표준 검증 패턴으로 효과적.
- CONTEXT.md decisions 블록을 PLAN.md 작성의 유일한 소스로 사용 — 결정 충실도 높고 scope creep 없음.
- Phase 경계 엄수: 이번 Phase에서 commands/ 삭제/plugin.json 수정 없이 skills/ 생성만 집중 → Phase 23이 명확한 범위를 이어받음.

- "완전 복사(exact copy)" 단일 원칙으로 14개 파일이 2개 커밋으로 완성됨 — 결정이 단순할수록 실행이 빠름.

### LEARNED
- `skills/` 디렉토리의 SKILL.md는 `Skill()` tool로 호출 불가. plugin.json "commands" 배열에 등록된 파일만 slash command로 접근 가능. sg-retro는 반드시 SKILL.md를 Read로 직접 읽어 실행해야 함.
- `git merge-base HEAD main`이 이미 main에 있으면 HEAD와 동일해 `BASE_SHA == HEAD_SHA` 에러 발생 — sg-review 단계에서 `HEAD~1` fallback 처리가 필요.
- sg-learn 명령이 구버전(hookify 라우팅)으로 로드됨 — 플러그인 캐시 갱신 지연 또는 로드 시점 고정 가능성. 현재 파일 내용과 로드된 내용이 다를 수 있으므로 실행 전 파일 직접 확인 습관 필요.

### LACKED
- Context 압축 중 gsd-plan-phase가 실행됐을 때 자동 복구 전략 없음 — PLAN.md 미생성 상황이 조용히 발생.
- SUMMARY.md가 생성되지 않아 회고 시 summary artifact가 없음.

### LONGED FOR

## DSPM — Decisions / Surprises / Patterns / Mistakes

### DECISIONS
- **완전 복사 전략 확정**: content 변경 없이 frontmatter + 블록 전체를 복사 — Phase 23에서 plugin.json 경로만 교체하면 전환 완료. 올바른 결정.
- **4 Plans, all wave:1**: 14개를 4그룹(sc-01~04)으로 분리. 실제로는 모두 wave:1이라 parallel 실행 없이 순차 처리됨 — GROUP_COUNT < 2 조건 충족.

### SURPRISES
- Context 압축이 gsd-plan-phase 실행 도중 발생 → PLAN.md 미생성. 롱 세션에서 context 압축은 예측 가능했으나 준비가 없었음.
- sg-learn 명령이 hookify를 참조하는 구버전으로 로드됨 — 플러그인 캐시 갱신 지연 추정.
- Codex 리뷰어가 `<done>false` 를 Important로 분류 → 훅이 읽지 않음 확인 후 Minor로 강등. 코드 확인 후 즉각 해결한 좋은 패턴.

### PATTERNS
- **파일 동일성 검증**: `diff -q source target` → OK/DIFF — 마이그레이션/복사 작업 표준 검증.
- **frontmatter + 블록 구조 검증**: `grep -c 'name:' + grep -c 'description:' + grep -c '<objective>'` 조합.
- **Phase 경계 준수 패턴**: commands/와 skills/ 공존 기간(Phase 22) → plugin.json 전환(Phase 23) → commands/ 삭제(Phase 23). 단계 간 독립성 유지가 충돌 없는 구현의 핵심.

### MISTAKES
- 구현 완료 후 PLAN.md `<done>false` 를 `true`로 업데이트하지 않음. 리뷰어가 Important로 플래그했으며, 확인 + 수정으로 해결됨. 향후 "구현 완료 체크리스트"에 포함 필요.
- SUMMARY.md 파일 생성 누락 — 구현 완료 후 phase summary 작성이 습관화되지 않음.
