## Milestone Summary

GSD → Superpowers → Hookify orchestration의 안정성·자기진단·세션 복원 강화. 세 가지 신뢰성 축을 한 milestone에 묶었다:

1. **sg-health** — 설치/훅/스키마/HANDOFF 무결성을 한 번에 진단하는 자기진단 명령
2. **Status accuracy** — sg-status의 STATE.md Phase 파싱 + display enum 매핑 + Next 라우팅 정확도 향상
3. **Session restore** — sg-start에서 기존 세션을 감지해 사용자에게 Resume / Start new milestone / Cancel 3-옵션을 제시

## Key Accomplishments

1. **sg-health 자기진단** — GSD/Superpowers/Hookify 설치 여부, hooks.json Stop/SubagentStop 등록, HANDOFF.md 5컬럼 스키마 무결성을 `[OK]`/`[WARN]`/`[FAIL]` 라인별로 출력. 부작용 없는 read-only 진단 (Phase 6).
2. **transcript_matcher 신호 정확도 보강** — `HOOKIFY_SIGNALS` 패턴 좁힘 + `IMPLEMENTATION_SIGNALS` 패턴 추가 (`finishing-a-development-branch`, `superpowers-implementation-complete`). false positive 제거 (Phase 6, 7 후속 수정).
3. **Storage vs Display enum 분리** — sg-status의 `STAGE_RAW`(7-state storage: init/gsd-plan/superpowers/review/hookify/ship/complete) ↔ `STAGE_DISPLAY`(축약 표시) 매핑 신설. `review` 표시는 `superpowers`로 통일하되 routing은 `/super-gsd:sg-learn`로 분리 (D-01~D-03). Phase 라인 파싱은 콜론 뒤 전체 trim, `\S+` 토큰 단일 캡처 폐기 (Phase 7).
4. **STATE.md Phase 파싱 회귀 방지** — `Phase: Not started` 같은 비숫자 라인에서 `Not`만 잘리던 버그 제거. BEGIN/END 주석 demarcate 형태로 sg-status에 블록을 lock하여 Phase 8 sg-start가 글자 그대로 인라인 복제 (Phase 7 D-07, Phase 8 D-03).
5. **sg-start 세션 복원** — STATE.md 존재 + `^Phase:` 캡처를 단일 트리거로 기존 세션 감지. Milestone / Phase / Stage / Last activity / Next 5-라인 emit 후 AskUserQuestion으로 Resume(emit-only) / Start new milestone(`Skill(gsd-new-milestone, args="")`) / Cancel(`Cancelled. No changes made.`) 3-옵션 분기. STATE.md 미감지 시 기존 `gsd-new-project + $ARGUMENTS` 그대로 (D-17 후방 호환). 세 옵션 모두 HANDOFF.md read-only — SESS-04 append-only 자연 충족 (Phase 8).
6. **워크플로우 라우팅 7-state 확장** — sg-ship/sg-complete가 HANDOFF에 `ship`/`complete` row를 명시적으로 기록. sg-status가 ship → next phase 또는 sg-complete 분기, complete → sg-new 분기를 정확히 제시 (v0.0.11).

## Overview

v1.1은 v1.0이 깐 GSD→Superpowers→Hookify 자동 인계 위에 **신뢰할 수 있는 진단·정확한 단계 인식·세션 복원**을 얹어, 사용자가 며칠 끊겼다 돌아와도 워크플로우가 자기 위치를 잃지 않도록 한다. 모든 phase는 단일 파일 수정 위주 (`commands/sg-*.md`)로 진행했으며, 별도 Python 헬퍼나 자동화 테스트 자산을 추가하지 않는 노선을 유지(Phase 6 D-04 + Phase 7 D-08 lock).

## Milestone Summary

**Decimal Phases:** 없음

**Key Decisions:**

- **Phase 6 D-04 (bash-only command files):** `commands/*.md` 파일 안의 bash 블록만 수정한다. 별도 Python/셸 헬퍼 모듈을 도입하지 않는다. v1.1의 후속 phase(7, 8)가 일관 유지.
- **Phase 7 D-01~D-03 (storage vs display enum 분리):** Storage 7-state는 라우팅 정확도, Display 4-state는 사용자 멘탈모델. `review`는 표시상 `superpowers`로 축약하되 라우팅은 `/super-gsd:sg-learn`으로 분리 — 사용자에게 보이는 stage 수는 적게, 워크플로우 정밀도는 잃지 않는다.
- **Phase 7 D-04~D-06 (STATE.md `Phase:` 라인 single source of truth):** sg-status의 phase 표시는 STATE.md 콜론 뒤 전체 문자열을 trim해 그대로 출력. ROADMAP.md 이름 룩업 폐기. `\S+` 단일 토큰 캡처 정규식 금지.
- **Phase 7 D-07 (인라인 복제 lock):** STATE.md Phase 파싱 블록을 BEGIN/END 주석으로 demarcate. Phase 8 sg-start가 같은 블록을 글자 그대로 인라인 복제하도록 사전 lock. drift 시 양쪽 동시 수정 강제.
- **Phase 8 D-13~D-16 (Resume/Start new milestone/Cancel):** "Fresh" 라벨 폐기. 3-옵션 명시. 세 옵션 모두 HANDOFF.md read-only — SESS-04 append-only 감사 로그 자연 충족.
- **Phase 8 D-17 (gsd-new-project 후방 호환):** STATE.md 미감지 시 기존 동작 그대로 `$ARGUMENTS` 패스스루. 기존 사용자 워크플로우 무영향.
