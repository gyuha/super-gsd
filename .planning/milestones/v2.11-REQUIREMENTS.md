# Requirements: v2.11 Add TDD workflow (sg-tdd)

**Milestone goal:** sg-execute와 sg-review 사이에 Superpowers `test-driven-development`를 오케스트레이션하는 red-green-refactor 검증 게이트 `sg-tdd`를 추가한다.

**Semantic note:** sg-tdd는 구현(sg-execute) 이후 단계이므로 고전적 "테스트 우선" TDD가 아니라 "TDD 준수 검증 게이트"다 — 테스트가 구현 없이는 실패하는지(의미 없는 통과 테스트 차단)를 확인하고 refactor 패스를 돌린다.

## v2.11 Requirements

### sg-tdd Command (TDD)

- [ ] **TDD-01**: 사용자가 `/super-gsd:sg-tdd [phase]`를 호출하면 sg-execute 산출물에 대해 Superpowers `test-driven-development` 스킬을 오케스트레이션하는 red-green-refactor 검증 루프를 실행한다
- [ ] **TDD-02**: sg-tdd는 Superpowers/GSD 파일을 수정하지 않고 외부에서 `test-driven-development` 스킬을 호출한다 (Non-invasive)
- [ ] **TDD-03**: sg-tdd는 검증 결과(테스트 red→green 확인, refactor 수행 여부)를 HANDOFF.md에 새 행으로 append한다 (append-only, `sg-tdd` stage 값)

### Pipeline Integration (PIPE)

- [ ] **PIPE-01**: config 플래그 `super_gsd.tdd_mode`(on/off)가 sg-tdd 단계 활성화를 제어한다 (부재 시 기본 off로 폴백)
- [ ] **PIPE-02**: sg-next와 sg-status가 sg-execute 완료 상태에서 `tdd_mode`가 on이면 sg-tdd로, off이면 sg-review로 라우팅하도록 양쪽 inline-replicated 라우팅 테이블을 동시 갱신한다 (D-07 규칙)
- [ ] **PIPE-03**: stop_hook/transcript_matcher가 sg-tdd 단계 진입·완료 신호를 감지해 다음 단계(sg-review)를 systemMessage로 안내한다

### Cross-Platform Mirror (MIRROR)

- [ ] **MIRROR-01**: `.agents/skills/sg-tdd/SKILL.md` 미러를 생성해 Codex/Gemini에서 `$sg-tdd`로 호출 가능하게 한다 (skills/ 원본과 pairwise sync)

### Documentation (DOC)

- [ ] **DOC-01**: README.md / README.ko.md Commands 표에 sg-tdd 행을 추가하고 파이프라인 서술(sg-plan→sg-execute→sg-tdd→sg-review)을 갱신한다
- [ ] **DOC-02**: CLAUDE.md 아키텍처/데이터 흐름 섹션에 sg-tdd 단계와 `super_gsd.tdd_mode` 플래그를 반영한다

## Out of Scope

- **고전적 테스트-우선 TDD(sg-execute 앞 배치)** — sg-tdd는 구현 이후 검증 게이트로 명시적으로 확정됨. 테스트-우선 배치는 향후 별도 검토
- **Superpowers `test-driven-development` 스킬 자체 수정** — Non-invasive 제약, 외부 오케스트레이션만
- **sg-execute 대체/제거** — sg-tdd는 sg-execute 다음 별도 단계이며 execute를 대체하지 않음
- **테스트 프레임워크별 어댑터 구현** — Superpowers TDD 스킬이 프레임워크 감지를 담당, super-gsd는 오케스트레이션만
