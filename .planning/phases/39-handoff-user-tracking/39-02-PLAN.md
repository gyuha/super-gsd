출력 형식 (CONTEXT.md 결정 B):
```
## 팀 현황

| 팀원 | 최근 Phase | 최근 Stage | 마지막 활동 |
| ---- | --------- | --------- | ---------- |
| Alice | 39-handoff-user-tracking | superpowers | 2026-05-28T10:00Z |
| Bob   | 38-...     | review     | 2026-05-27T18:30Z |
```

데이터 소스 우선순위:
1. HANDOFF.md의 `$7` (User) 컬럼이 있고 `-` 가 아닌 행 → 팀원별 최신 행 1개
2. 위 행이 없으면 → `git log --format="%an|%ai|%s"` 최근 커밋에서 작성자 집계
3. 둘 다 없으면 → "팀 이력이 없습니다." 메시지 출력

awk 컬럼 인덱스 (6열 HANDOFF 행 기준):
`| ts | phase | from | to | plan_hash | user |`
→ $1=empty, $2=ts, $3=phase, $4=from, $5=to, $6=plan_hash, $7=user, $8=empty
따라서 User 컬럼은 반드시 $7을 사용한다.

macOS 호환성 주의:
- `awk -F'|'` 사용 (CLAUDE.md 컨벤션)
- `grep -P` 금지, `-E` 사용
- `sort -t'|'`로 파이프 구분 정렬

sg-status 기존 동작 (`--team` 없을 때) 완전 유지 — 새 로직은 `$ARGUMENTS`에 `--team`이 있을 때만 실행 후 exit.
