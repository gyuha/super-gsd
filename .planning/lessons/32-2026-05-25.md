### Draft sg-rules

**rule: warn-agents-read-comment-in-bash**
```yaml
name: warn-agents-read-comment-in-bash
enabled: true
event: file
conditions:
  - field: file_path
    operator: regex_match
    pattern: "\.agents/skills/.*SKILL\.md"
  - field: new_string
    operator: regex_match
    pattern: "#\\s*Read .*\\.planning"
action: warn
```
> `.agents/skills/` SKILL.md에서 `# Read .planning/...` 패턴 감지 — bash 주석이므로 실행 시 무시됨. `#` 없이 `Read .planning/...`으로 작성해야 함.

**rule: warn-node-process-env-arguments**
```yaml
name: warn-node-process-env-arguments
enabled: true
event: bash
pattern: "process\\.env\\.ARGUMENTS"
action: warn
```
> `node -e` 인라인 JS에서 `process.env.ARGUMENTS` 사용 감지 — 셸 변수는 node 자식 프로세스에 자동 export되지 않음. `process.argv[2]` + `-- "$ARGUMENTS"` 패턴을 사용할 것.

**rule: warn-read-inside-bash-fence**
```yaml
name: warn-read-inside-bash-fence
enabled: true
event: file
conditions:
  - field: new_string
    operator: regex_match
    pattern: "```bash[^`]*Read \\.(planning|agents)"
action: warn
```
> bash 코드 펜스 안에 `Read .planning/` 또는 `Read .agents/` 지시문이 혼재 — prose 레벨로 분리해야 함.
