    1. `jq 'del(.commands)' /Users/gyuha/workspace/super-gsd/.claude-plugin/plugin.json > /tmp/plugin_tmp.json`

    주의: Write 도구로 JSON을 직접 작성하는 경우 RESEARCH.md의 "plugin.json 최종 상태" 코드 예시의 정확한 필드 목록과 값을 그대로 사용한다. version 필드(현재 0.0.28)를 변경하지 않는다.

    검증: `jq '.commands // "null"' .claude-plugin/plugin.json` 결과가 `"null"`이어야 한다.
  </action>
  <verify>
    <automated>cd /Users/gyuha/workspace/super-gsd && jq '.commands // "null"' .claude-plugin/plugin.json | grep -q '"null"' && echo "PASS: commands 키 없음" || echo "FAIL"</automated>
  </verify>
  <acceptance_criteria>

    - `jq '.skills' .claude-plugin/plugin.json` 출력이 `"./skills/"`이다
    - `jq '. | keys' .claude-plugin/plugin.json` 출력에 `commands`가 포함되지 않는다
    - `jq '. | keys' .claude-plugin/plugin.json` 출력에 `name`, `version`, `skills`, `keywords`가 포함된다
    - `jq . .claude-plugin/plugin.json` 실행 시 파싱 오류가 없다
  </acceptance_criteria>
