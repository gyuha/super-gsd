    **_extractHookifyOutput(transcriptPath) -- mirror stop_hook.py:65-77:**

    Note B-1 alignment: while plan 28-01's `_splitlines` helper applies to `detectSignal`, this function (stop_hook's `_extractHookifyOutput`) uses a different Python primitive (`readlines()`, which DOES preserve terminators). The lookbehind split below is the parity-correct port — it does NOT need the trailing-empty pop because the lookbehind preserves the line-ending bytes inline.

    ```
    function _extractHookifyOutput(transcriptPath) {
      try {
        const content = fs.readFileSync(transcriptPath, 'utf-8');
        return _joinLastNLinesWithTerminators(content, 200, ['## Lessons', '## Patterns', '## Hooks Generated', 'hookify complete']);
      } catch (e) {
        return '';
      }
    }

    function _joinLastNLinesWithTerminators(content, n, markers) {
      // Replicate Python: lines = f.readlines(); recent_text = ''.join(lines[-n:]); then rfind each marker.
      // readlines() returns lines INCLUDING trailing \n (Python preserves terminators per element).
      // JS lookbehind split (?<=\n) yields elements each ENDING with \n, matching readlines.
      const parts = content.split(/(?<=\n)/);
      const recent = parts.slice(-n).join('');
      for (const marker of markers) {
        const idx = recent.lastIndexOf(marker);
        if (idx !== -1) return recent.slice(idx);
      }
      return recent;
    }
    ```

    CRITICAL PARITY GOTCHA (the most subtle in Phase 28):
    - Python `readlines()` returns a list where each element INCLUDES the trailing `\n` from the original file.
    - Python `''.join(lines[-200:])` concatenates without inserting separators, preserving the original byte layout.
    - JS `content.split('\n')` STRIPS terminators -- joining with `'\n'` then re-adds them, but the final line will have an EXTRA `\n` if the file ended with `\n` (or a MISSING `\n` if it didn't), breaking byte-for-byte parity.
    - SOLUTION: the lookbehind split `(?<=\n)` yields elements that each END with `\n` (matching Python readlines). Node 18+ supports lookbehind.
