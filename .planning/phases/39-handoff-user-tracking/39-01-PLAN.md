수정 패턴 — 모든 HANDOFF.md append 위치에 동일하게 적용:

기존 (5열 echo):
```bash
echo "| $TS | $PHASE_SLUG | $FROM_STAGE | {stage} | - |" >> "$HANDOFF_FILE"
```

신규 (6열 echo):
```bash
GIT_USER=$(git config user.name 2>/dev/null || echo "-")
[ -z "$GIT_USER" ] && GIT_USER="-"
echo "| $TS | $PHASE_SLUG | $FROM_STAGE | {stage} | - | $GIT_USER |" >> "$HANDOFF_FILE"
```

헤더 초기화 블록도 동일하게 6열로 변경:
```bash
printf '| Timestamp | Phase | From | To | Plan Hash | User |\n| --- | --- | --- | --- | --- | --- |\n' > "$HANDOFF_FILE"
```

주의: sg-execute(skills/)의 idempotency 체크에서 `$6`이 Plan Hash를 읽는 기존 awk가 있다.
User 컬럼 추가로 Plan Hash는 `$6` → `$6` 그대로(열이 밀리지 않음—Plan Hash가 5번째 pipe 필드이므로
`awk -F'|' '{gsub(/ /,"",$6); print $6}'`는 6번째 pipe-delimited field = 5번째 내용 컬럼 = Plan Hash).
단, User 컬럼 추가 후 7번째 pipe 필드가 User가 된다. awk $6 = Plan Hash는 열이 추가되어도 영향 없음.
</context>
