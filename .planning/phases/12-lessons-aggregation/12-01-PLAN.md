    - `python3 hooks/lessons_ranker.py .planning/lessons/` 실행 시 JSON lines 출력 (score 내림차순, 기본 top 5)
    - `--top 3` argument로 N 조정 가능
    - score 공식: `score = 0.4 * freq + 0.4 * recency + 0.2 * severity` (per D-02)
      - freq: lessons 파일에서 동일 카테고리(## 섹션 헤더) 등장 횟수
      - recency: `1 / (1 + days_ago)` (파일 mtime 또는 파일명 날짜 기준)
      - severity: high=1.0, medium=0.5, low=0.2, 없음=0.3
    - stdout 형식: `{"pattern": "...", "score": 0.xx, "source": "filename:section"}`
    - `--archive --milestone v1.2` 모드: `.planning/lessons/*.md`를 읽어 `.planning/milestones/v1.2-LESSONS.md`로 복사 (원본 유지). 이미 파일이 있으면 덮어쓰지 않고 warn 출력.
    - lessons 파일이 없으면 빈 JSON array 출력(ranking mode) 또는 warn-only(archive mode)
    - 실패 시 exit code 1, stderr에 에러 메시지
  </behavior>
  <action>

    3. `compute_scores(entries, today)` — D-02 공식 적용. freq는 같은 카테고리명 전체 등장 횟수(파일 경계 무관). recency는 파일명에서 YYYY-MM-DD 추출 시도, 실패 시 os.path.getmtime 사용.
    4. `ranking_mode(args)` — top-N JSON lines 출력
    5. `archive_mode(args)` — milestone 버전 받아 .planning/milestones/ 디렉토리 생성 후 v{X}-LESSONS.md 작성. 기존 파일 존재 시 `[warn] archive already exists: ...` 출력 후 skip.
    6. `main()` — argparse로 `paths(positional, nargs='*')`, `--top N`, `--archive`, `--milestone vX.Y` 파싱. archive면 archive_mode 호출, 아니면 ranking_mode 호출.
