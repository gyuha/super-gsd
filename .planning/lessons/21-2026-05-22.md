## Lens: Five Whys
_Captured: 2026-05-22T14:23:16Z_

### Problem Statement
- Workflow 다이어그램 optional branch 표기 누락 — Plan에도 없었고 코드 리뷰까지 발견하지 못함

### Why 1
- Plan에 Workflow 다이어그램 표기 업데이트 태스크가 없었다

### Why 2
- Plan 작성자가 Workflow 다이어그램이 업데이트 대상임을 인식하지 못했다

### Why 3
- 명령 추가 시 업데이트해야 할 문서 목록이 컴포넌트 수준(파일)에만 정리되어 있다

### Why 4
- GSD Plan 템플릿에 '파급 문서 업데이트 여부' 체크 항목이 없다

### Why 5
- GSD가 코드 중심 템플릿이라 시스템 수준 문서 연쌓은 사용자 체재 (의도된 한계)

### Root Cause
- GSD Plan 템플릿은 코드 파일만 추적하므로, 코드 변경이 유발하는 시스템 수준 문서(다이어그램, Workflow Guide) 업데이트는 사용자가 별도로 인식해야만 포함됨 — 이를 보완할 super-gsd 프로젝트 수준 체크리스트나 컨벤션이 없었음

| P1 | CLAUDE.md 컨벤션에 "새 sg-* 명령 추가 시 Workflow 다이어그램 업데이트" 명시 | CLAUDE.md Conventions 섹션에 항목 추가 |
| P2 | Plan에 "파급 문서 업데이트 여부 확인" 태스크 템플릿화 | 향후 sg-* 명령 추가 phase Plan에 해당 체크 항목 수동 포함 |

### Draft sg-rules

**Rule 1: warn-state-phase-awk-token**
```yaml
name: warn-state-phase-awk-token
enabled: true
event: bash
conditions:
  - field: command
    operator: regex_match
    pattern: "STATE\\.md.*awk.*print.*\\$1|awk.*print.*\\$1.*STATE\\.md"
action: warn
```
경고: `awk '{print $1}'`로 STATE.md Phase 번호 파싱 시 `Phase: Phase 22 (...)` 형식에서 실패.  
대신 `grep -E '^Phase:' | sed -E 's/^Phase:[[:space:]]*//' | awk '{print $1}'` 파이프라인을 사용할 것.

**Rule 2: warn-plugin-json-skills-field**
```yaml
name: warn-plugin-json-skills-field
enabled: true
event: file
conditions:
  - field: file_path
    operator: regex_match
    pattern: "plugin\\.json$"
  - field: new_string
    operator: not_regex_match
    pattern: '"skills"'
action: warn
```
경고: plugin.json을 수정할 때 `"skills": "./skills/"` 필드가 없으면 슬래시 명령 자동 등록이 깨진다.
